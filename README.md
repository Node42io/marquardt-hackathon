# Marquardt Ultrasonic Flow Sensor — Market Analysis

React app that renders the Clayton market-analysis report for the Marquardt ultrasonic flow sensor. All data lives in JSON under `src/data/` — generated by the extractor from the markdown source in `../sections/`.

## Pages

- **Product Decomposition** — Christensen abstraction, technology classification, functional promise, features, specs, constraints, BOM.
- **Functional Promise** — technology + UNSPSC commodity promise.
- **Constraints** — 12 absolute + mitigable constraints with per-constraint sources.
- **Home Market** — tabbed: Competition / JTBD / Value Network / Bill of Materials / Kano / Compatibility / Alternatives (NAICS 333415).
- **New Market Discovery** — NAICS explainer, 16 candidates, architecture distance, 6-factor composite ranking for all 9 markets.
- **New Market Analysis** — tabbed per market × analysis: JTBD (ODI matrix + Burleson pyramid + Job Map + stakeholders) · Value Network · Bill of Materials · Kano · Compatibility · Alternatives.

## Stack

- Vite + React 19 + TypeScript
- React Router (HashRouter for static hosting)
- Tailwind v3 + the dark-theme CSS ported from the original report
- `@floating-ui/react`-backed Tooltip / Popover primitives

## Develop

```bash
npm install
npm run dev                # localhost:5173
npm run build              # production build into dist/
```

## Data pipeline

```bash
node scripts/extract-data.mjs       # rebuild all JSON from ../sections/*.md
```

See `DATA_SCHEMA.md` for the canonical JSON shape and `DATA_AUGMENTATION_LOG.md` for every post-extraction augmentation and the upstream fix it implies.

## Deploy — GitHub Pages

Pushing to `main` triggers the Pages workflow in `.github/workflows/deploy.yml`. The site is served at:

```
https://node42io.github.io/marquardt-hackathon/
```

Build-time base path is injected via the `VITE_BASE` env var so you can also deploy to any other path.
